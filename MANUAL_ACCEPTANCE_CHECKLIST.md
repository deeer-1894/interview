# 页面级手工验收清单

适用范围：`工作区列表 / 会话页 / 复盘弹层 / 策略 tab / 追问树 / 阶段视图 / 画像`

目标：确认最近一轮产品改造在真实页面上“显示正确、交互稳定、路径清晰”，并和后端 review snapshot / event 数据保持一致。

## 验收前准备

1. 启动后端与前端开发环境，确保页面可正常打开。
2. 先执行一次接口级 smoke test，确保基础数据链路可用：

```bash
bash /Users/shiyi/mockinterview/scripts/smoke_test.sh
```

3. 准备至少 3 类工作区样本：
   - `进行中`：有正在运行或刚恢复的会话
   - `已完成`：有 review snapshot、scorecard、trace、profile 的会话
   - `命中过历史弱项`：summary 里有 `historicalWeaknessesHit`

4. 建议浏览器窗口同时覆盖：
   - 桌面宽屏
   - 窄一些的笔记本宽度

## 一、工作区列表

### 1. 列表排序

操作：
1. 打开首页左侧工作区列表。
2. 对比多个工作区的排序。

预期：
1. `置顶` 工作区优先出现。
2. `等待澄清`、`运行中 / 恢复中` 的工作区应靠前。
3. `最近命中历史弱项`、`压力阶段 / 收尾阶段` 的工作区相比普通已完成项更靠前。
4. 同级数据下再按更新时间排序。

重点观察：
- 排序变化应稳定，不应在静置时自己跳动。
- 列表刷新后不应把当前选中的工作区切走。

### 2. 列表摘要

操作：
1. 观察每个列表项的 badge 和第二行摘要。
2. 分别点开一个普通已完成会话和一个命中过弱项的会话。

预期：
1. badge 能表达主要状态，例如 `已完成`、`已评分`、`压力阶段`、`收尾`、`已置顶`。
2. 第二行摘要能表达最近策略，而不是只重复标题。
3. 命中过历史弱项的工作区，会出现类似“再次命中 xxx”的摘要。
4. 不同工作区摘要应和实际复盘内容一致。

### 3. 切换与回到总览

操作：
1. 点击一个工作区进入会话。
2. 再点击左侧“工作区”标题区回到总览。
3. 再点另一个工作区。

预期：
1. 点击工作区会进入对应会话，不会错进其他 run。
2. 点击“工作区”标题区会回到总览首页。
3. 从总览再进入其他工作区时，页面状态清晰，没有残留上一个会话的复盘内容。

## 二、会话页稳定性

### 4. 静置稳定性

操作：
1. 进入一个已有会话。
2. 不点击、不输入、不滚动，静置观察一段时间。

预期：
1. 页面不会自动跳到同会话下的其他历史记录。
2. 页面不会自动切到模型早先思考输出的位置。
3. 左侧选中项保持不变。
4. 顶部状态条、输入框、消息区不应无故抖动。

### 5. 滚动与输入

操作：
1. 在一场较长的会话里手动向上滚动查看历史消息。
2. 等待系统继续输出，观察是否自动拉回底部。
3. 在输入框里连续输入长句。

预期：
1. 手动上滚后，不会强制把阅读位置拉回最新消息。
2. 只有用户明确回到底部或主动发送后，才继续跟随输出。
3. 输入框打字应明显顺畅，不应出现整页卡顿。

## 三、复盘弹层总体验

### 6. 打开入口与默认 tab

操作：
1. 在运行中会话点击 `会话配置 / 复盘`。
2. 在已完成会话点击同一入口。

预期：
1. 运行中如果已有决策审计，默认优先打开 `策略` tab。
2. 已完成会话如果已生成评分，默认能看到 `评分` 或最有信息量的 tab。
3. tab 标题和 hint 文案与数据状态一致，例如 `已生成`、`等待结果`、`等待链路`、`等待沉淀`。

### 7. 摘要区

操作：
1. 打开一个已完成且带历史弱项命中的复盘。

预期：
1. 顶部摘要能看到当前阶段、核心说明、必要时的历史弱项命中提示。
2. 摘要内容和下方 `评分 / 策略 / 追问树 / 画像` 数据不冲突。

## 四、策略 Tab

### 8. 最近一轮策略

操作：
1. 打开 `策略` tab。

预期：
1. 能看到最近一轮的 `reason` 和 `explanation`。
2. 能看到当前阶段、轮次、压力等级等状态信息。
3. 能看到弱信号、强信号、画像重点、下一步 focus。
4. 如果会话刚开始，允许 `history` 很短甚至为空，但 explanation 仍应存在。

### 9. 跨轮策略历史

操作：
1. 在至少两轮以上的会话中查看 `策略` tab。
2. 依次浏览每轮策略记录。

预期：
1. 每一轮都能看到对应的策略原因和 explanation。
2. 若该轮命中了历史弱项，相关说明应能从策略或关联 trace 中看出来。
3. 点击策略历史项能跳到对应问答节点或相邻内容。

## 五、追问树 Tab

### 10. 节点信息

操作：
1. 打开 `追问树` tab。
2. 检查首个节点及后续节点。

预期：
1. 节点至少应包含：问题、轮次、阶段、reason/explanation 中的关键信息。
2. 若节点命中画像弱项，应显示 `命中历史弱项` 或同类提示。
3. `focusHits`、`weakSignals`、`strongSignals` 应以可读方式展示，而不是原始 JSON 感。

### 11. 搜索 / 导出 / 浏览

操作：
1. 在追问树搜索一个关键词。
2. 试用导出 SVG。
3. 展开浏览多个节点。

预期：
1. 搜索结果能正确过滤节点。
2. 导出 SVG 正常，内容与当前追问树一致。
3. 节点层级和路径清楚，不会出现错乱或空白大片区域。

## 六、阶段视图

### 12. 阶段节奏

操作：
1. 打开 `阶段` 视图。

预期：
1. 能看出从热身、深入、压力、收尾等阶段的推进节奏。
2. explanation 和历史命中应能体现在阶段总结里。
3. 历史弱项命中次数统计应和追问树节点数大致一致。

## 七、画像 Tab

### 13. 画像沉淀

操作：
1. 打开 `画像` tab。
2. 检查维度、最近技能、最近人格、最近更新等信息。

预期：
1. `interviewCount`、`dimensions`、`stableStrengths`、`recurringGaps`、`recommendedFocus`、`recentChanges` 显示正常。
2. 若有本场再次命中的历史弱项，应在画像页显式出现。
3. 没有画像数据时，应显示合理的空状态，而不是留白或报错。

## 八、评分 Tab

### 14. 评分结果

操作：
1. 打开 `评分` tab。

预期：
1. 可看到总评 summary。
2. 若有 `dimensionScores`，应逐项展示分数和说明。
3. strengths / gaps / improvements / studyPlan 各分区表现正确。
4. 对于极短会话，即使没有完整 scorecard，也不应导致整个复盘弹层异常。

## 九、跨视图一致性

### 15. 一致性检查

操作：
1. 选择一个“命中过历史弱项”的会话。
2. 依次查看：左侧列表摘要、复盘摘要、策略 tab、追问树、画像。

预期：
1. 左侧摘要里的“再次命中 xxx”能在复盘摘要中找到对应项。
2. 追问树里至少有节点体现 `profileHit / focusHits`。
3. 画像页的“本场再次命中的历史弱项”和 summary 一致。
4. 策略里的 explanation 与追问树、阶段视图的 explanation 不冲突。

## 十、异常与回归关注点

### 16. 重点回归

每次改动后建议至少回归以下问题：
1. 会话静置时是否自动跳到其他记录或旧输出位置。
2. 输入框是否再次变慢。
3. 左侧工作区排序是否在刷新后异常变化。
4. 复盘弹层是否会打开空 tab 或默认 tab 错误。
5. `策略 / 追问树 / 画像` 是否只在事件流里有，刷新后丢失。
6. conversation detail / task create 接口是否再次泄露 `modelConfig.apiKey`。
